# 决策记录（Decision Records）

> 每个重大决定一个文件。一年后你不会记得为什么定的，但文件会。

---

## 为什么要写这个

早期项目最大的浪费是：

- **两人对同一件事记忆不一致**，回头才发现做反了
- **三个月后忘了为什么这么定**，重新讨论一遍浪费两小时
- **新成员加入**，没人能完整解释"为什么我们做 X 而不是 Y"

这个文件夹就是项目的"长期记忆"——所有方向性决定都写在这里。

---

## 什么样的决定要记

**必须记**：
- 产品定位 / 核心交互模型的变化
- 技术栈 / 架构选型
- 商业模式 / 定价 / 计费方式
- 品牌相关（产品名、配色、Voice）
- 重要的人员 / 流程决定（外包、合伙人股权、招人）

**可选记**：
- 一些工具选择（用 Linear 还是飞书）
- UI / 组件库选择

**不用记**：
- 日常代码细节
- 一次性 bug fix
- 措辞调整

---

## 文件命名约定

```
NNNN-短标题.md
```

- `NNNN` = 4 位数字编号，按时间顺序，从 `0001` 开始
- 短标题 = 用英文 kebab-case，方便排序和 grep
- 例子：
  - `0001-c3-interaction-model.md`
  - `0002-managed-api-not-byok.md`
  - `0003-pricing-199-per-month.md`
  - `0004-no-chrome-extension-in-v1.md`

---

## 每个决策文档的格式

复制 `_template.md` 来写新决定。结构：

```markdown
# NNNN · 决策标题

- **日期**: 2026-06-25
- **状态**: 提议 / 已采纳 / 已废弃 / 被 NNNN 取代
- **决策人**: Joe / 合伙人 / 共同

## 背景
（这件事为什么需要决定？发生了什么？）

## 候选方案
（考虑过哪几种？各自的取舍）

## 决定
（最终选了哪个）

## 理由
（为什么这个最好？关键的判断依据）

## 后果
（这个决定带来什么改变？需要做什么）

## 后续
（什么情况下应该回来重新审视这个决定？）
```

---

## 状态流转

| 状态 | 含义 |
|---|---|
| **提议** | 正在讨论，还没拍板 |
| **已采纳** | 正在执行 |
| **已废弃** | 不再有效（说明原因） |
| **被 NNNN 取代** | 有新决策取代了它（引用新决策编号） |

**重要**：被取代的决策**不要删除**，留作历史记录。在文档顶部加一行链接到取代它的新决策即可。
