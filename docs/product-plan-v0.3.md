## 零、一页摘要

### 一句话定义

> izanoffer 是一个中国学生能直接用的留学专用 AI 顾问。它既懂留学（顶级顾问能力），也能看懂任何学校官网（实时翻译解读），让每个答案都有官网兜底。

### 公式

> **Claude / GPT 顶级能力  ×  中国学生访问壁垒  ×  留学场景深度**

三个变量缺一不可。我们把全世界最好的 AI 顾问能力，包装成中国学生能直接用的留学专用工具。

### 核心交互（C3 模型）

- **左边**：学校官网原文（纯净阅读视图，不动）
- **右边**：AI 在对话框里翻译、总结、解读、给建议
- **桥接**：划词即解读、URL 直接粘贴、引用反向高亮
- **信任锚**：右上角『↗ 在原网站打开』永远在场

### 产品定位的三个对手

| 对手 | 他们的弱点 | 我们的优势 |
|---|---|---|
| 传统留学中介 | 3-10 万收费、顾问轮换、文书造假 | 199/月、7×24、永远耐心 |
| Claude / ChatGPT 原生 | 中国用不了、不懂留学、不会引官网 | 本地化 + 垂直 + 双栏交互 |
| iOFFER 等 AI 留学产品 | 想替学生做事 → 高风险、低信任 | 帮学生看懂 → 低风险、高信任 |

### v0.1 → v0.3 的关键转变

| 维度 | v0.1（旧） | v0.3（现在） |
|---|---|---|
| 产品理念 | AI 替你做留学 | AI 帮你看懂留学 |
| AI 角色 | 代理人（写文书、填表） | 顾问 + 翻译者（解读、建议） |
| 核心交互 | 对话框聊天 | 双栏：官网原文 + AI 解读 |

| 合规姿态 | AI 写文书有政策风险 | AI 不动笔，规避所有学校政策 |
| 商业模式 | 中介替代 | 顶级顾问 + 官网翻译者 |

## 一、产品理念

### 1.1 三个核心洞察

#### 洞察一：中国学生用不了 Claude

中国 60+ 万出国留学生事实上访问不了 Claude / GPT —— 需要 VPN、国际信用卡、英文界面、不会注册。但他们最需要这类顶级模型的能力：看英文官网、深度思考、解读复杂材料。我们把它包装成中国学生能直接用的留学产品，本身就是一个套利窗口。

#### 洞察二：留学是高决策成本场景，AI 必须「看得见出处」

留学是几万到几十万的不可逆决策——错一个 deadline、信错一个录取要求，可能耽误一年。学生不敢相信「AI 单口相声」。所以我们把官网放在学生眼前——AI 给建议时，原文同时可见。这是其他通用 AI 产品给不了的体验。

#### 洞察三：顾问 + 解读，是市场上没人占据的位置

传统中介只做顾问（高客单、低规模）。iOFFER 这类 AI 产品想替学生做事（高合规风险）。Claude / ChatGPT 是通用工具（不懂留学）。我们站到第三个位置：垂直留学的顾问 + 官网解读，既懂又透明。

### 1.2 与通用大模型的根本区别

| 维度 | Claude / ChatGPT 原生 | izanoffer |
|---|---|---|
| 知识范围 | 训练截止前的通用网络 | 持续更新的学校 / 项目 / 政策结构化数据库 |
| 专业深度 | 本科生水平泛泛之谈 | 资深留学顾问 + 招生委员视角 |
| 中国学生语境 | 无（默认美国本土学生） | 双非、GPA 算法、F-1 行政审查等都是默认前提 |
| 官网阅读 | 通用 webfetch，不懂学校结构 | 学校官网专用 pipeline + 自动定位关键章节 |
| 呈现方式 | 纯对话 | 双栏：左原文 + 右解读 + 划词联动 |
| 中文支持 | 可以中文，但默认体验是英文 | 界面 / 顾问语气 / 案例库全中文原生 |
| 合规性 | 无（学生用 Claude 写文书会被 AI 检测） | AI 不动笔，全程「解读 + 建议」，不触发学校 AI 政策 |

### 1.3 关键设计原则

- **对话框是入口，技能包是壁垒。** UI 极简，但每问一个问题，背后是几十个 skill、几万条结构化数据、几百个 prompt 在协作。
- **事实在左，判断在右。** 官网原文神圣不可侵犯，AI 完全在右边对话框里说话。心智清晰。
- **AI 不动笔，只解读。** 不写文书、不填表、不替学生做不可逆的事。文书最多给思路反馈，不直接生成段落。
- **底层只为留学服务。** 永远不接电商、求职、移民、海外购房等周边业务。做深不做广。

## 二、核心交互模型（C3）

### 2.1 双栏布局

产品的灵魂时刻被压缩成两件事：

| 区域 | 内容 | 性质 |
|---|---|---|
| 左栏 · 官网视图 | 学校 / 官方机构 / 表单的原文（快照渲染） | 事实层，AI 不修改 |
| 右栏 · AI 顾问对话 | 翻译、总结、引用左边、对照学生背景的建议 | 解读层，AI 全部在这里 |
| 桥接 | 划词解读、URL 粘贴、引用反向高亮 | 左右真正联动 |

### 2.2 一次对话的完整流程

以学生输入「米兰理工有多少专业」为例：

- **Step 1** 意图识别 → Skill Router 判断属于「学校项目信息查询」
- **Step 2** 数据源决策 → 应该读 polimi.it 的 programmes 页面
- **Step 3** 抓取 + 渲染 → 后端抓官网内容，渲染到左栏 viewer
- **Step 4** AI 主动开场 → 右栏自动生成翻译 + 总结 + 留学视角解读
- **Step 5** 引用联动 → AI 答案里的 P1/P2/P3 角标 → 点击 → 左栏对应段落高亮闪烁 + 滚动定位
- **Step 6** 学生追问 → 划选英文段落 → 浮出「让 AI 解读这段」按钮 → 选中文本作为引用塞进对话框 → AI 即时翻译解读

### 2.3 四种左侧形态

左栏不只是「学校官网 viewer」，是「权威源 viewer」。源的类型由问题决定：

| Pattern | 适用场景 | 信任锚 | 视觉形态 |
|---|---|---|---|
| **A · 官方网站阅读** | 学校 / 政府机构 / 移民局 / 教育部官网 | 单一权威源 | 纯净英文原文 + 跳转真站按钮 |
| **B · 多源对照地图** | 无单一权威，需综合（租房平台对比、奖学金多家来源） | 多源 + AI 综合 | 结构化对比表 / 平台横向卡片 |
| **C · 案例库 / 经验墙** | 官方未覆盖的实战（避坑、文书参考、签证 case） | 集体经验 | 案例卡片墙 + 用户众包数据 |
| **D · 表单截图引导** | DS-160 / Préfecture / CommonApp 等填表类 | 真实表单截图 + 逐项注释 | 截图 + 注释气泡 |

### 2.4 入口与扩展

用户进入双栏对话的入口有 4 种，全部触发同一个核心交互：

| 入口 | 体验 |
|---|---|
| 问问题（默认） | AI 自动判断 + 抓对应官网 |
| 粘贴 URL | AI 直接抓那个 URL + 进入双栏 |
| 上传 PDF | AI 解析 PDF + 显示在左栏 |
| 截图官网某段 | OCR + 进入双栏 |

> 核心交互不变，入口扩展。这让产品同时拥有「懂留学的深度」+「任何留学相关材料都能扔过来的广度」。

## 五、边界模型（三圈 + 拒绝话术）

> **克制比扩张更值钱。一上来什么都做的产品，没有一个能做深。**

### 5.1 三圈定义

| 圈 | 范围 | 处理 |
|---|---|---|
| 🟢 **绿区** | 留学专属决策（选校、文书、签证、申请、居留） | 深度回答 + 调用左栏 + 完整 skill |
| 🟡 **黄区** | 留学场景中的通用事（机票、行前购物、银行卡） | 给「留学生视角」的简短答 + 推荐通用工具 |
| 🔴 **红区** | 与留学身份无关（餐厅推荐、代码、投资） | 礼貌拒绝 + 推荐合适工具 |

**判断标准**：「中国留学生」这个身份是否构成关键变量？

- 是 → 绿区或黄区
- 否 → 红区

### 5.2 拒绝话术（红区示例）

**比价机票**：
> 比价机票推荐你用 Skyscanner / 携程国际版，专业的事交给专业的工具。不过留学生有几个专属知识点我可以帮你：学生票优惠（美联航有 +1 件行李）、提前 60-90 天订最划算、转机国家签证要求。要不要继续聊这些？

**餐厅推荐**：
> Google Maps / 小红书 / TripAdvisor 更专业。但如果你问的是「刚到米兰第一周怎么解决吃饭和省钱」，这是个留学生话题，我可以帮。

**投资理财**：
> 我不是持牌投资顾问。但如果你问的是「F-1 身份开美股账户的合规问题、报税怎么处理」，这就是我的事。

**写代码**：
> 代码问题 Claude / ChatGPT 答得比我好。我只在留学相关的代码场景深耕——比如 CS PhD 申请的 portfolio 怎么准备、美本 OA 题型规律。

### 5.3 拒绝的三个原则

- 不傲慢——不说「我不答」
- 不弱势——不说「我答不出来」
- 永远给备选——不让用户被卡住

### 5.4 隐藏的逃生通道（订阅版）

红区问题底部可以加一个不起眼的按钮：「💬 这个问题超出我的能力，要不要用通用 Claude 答一次？」点击用底层模型回答，明确告知不是 izanoffer 的能力。这样重度用户不会被卡走，但默认体验始终是「专业留学顾问」。

## 六、准确性与信任架构

> **LLM 不存事实，LLM 只做推理。所有事实型答案必须从知识库实时检索。**

### 6.1 五层防御

| 层 | 做什么 | 实现 |
|---|---|---|
| L1 · RAG 优先 | 事实型数据全部从结构化知识库取，不靠 LLM 记忆 | 学校 / 政府数据库 + Embedding 检索 |
| L2 · 数据源分级 | T1 直接信任 / T2 结构化采集 / T3 标注来源 / T4 不采用 | 数据采集 pipeline |
| L3 · 答案分层标注 | 事实附来源 + 时效；推断标注「基于 N 案例」；判断说「我的建议」 | 答案 schema + UI 渲染 |
| L4 · LLM 行为约束 | 系统 prompt 强制「不知道就说不知道」；高风险问题走专用流程 | Prompt 工程 + 关键词触发器 |
| L5 · 反馈与人工兜底 | 每条回答带「信息有误」按钮；签证 / 文书重大决定走人工 review | 运营流程 |

### 6.2 数据源分级标准

| Tier | 来源 | 信任度 | 更新频率 |
|---|---|---|---|
| T1 | 学校官网 / 教育部 / USCIS / 移民局 | 100% 信任 | 每周抓取 + 变更告警 |
| T2 | U.S. News / QS / Niche / Common App | 高度信任 | 每月抓取 |
| T3 | Reddit / GradCafe / 一亩三分地 / 知乎 | 信号但需标注 | 实时索引 |
| T4 | 机构软文 / 商业留学博客 | 不采用 | — |

### 6.3 用户看得见的信任元素

- 每条事实型回答附「来源 + 抓取时间」标签
- AI 引用编号（P1/C2/U3）点击高亮左栏对应段落
- 右上角『↗ 在原网站打开』永远可用，跳真官网核对
- 高风险问题（签证、文书定稿）触发「建议人工复核」提示
- AI 政策实时跟踪：Caltech / Cornell / 墨大等校的最新文书政策

## 七、模型与计费策略

### 7.1 MVP 阶段 = 100% Managed API

用户付订阅费，我们后端调用模型。不做 BYOK（学生注册海外 API 转化率不到 5%）。

### 7.2 智能模型路由

| 场景 | 用什么模型 | 理由 |
|---|---|---|
| 意图识别 / 简单 FAQ | 国产模型（DeepSeek / 通义 / 豆包） | 1/10 成本，足够 |
| 官网解读 / 翻译总结 | Claude Sonnet | 中英文 + 长文档能力业界最强 |
| 复杂选校推理 / 顾问对话 | Claude Opus / GPT-5 Pro | 高质量推理 |
| 案例库检索 / 嵌入 | 便宜 embedding 模型 | 向量检索量大 |

### 7.3 成本武器

- **Prompt Caching**（Claude 原生支持，能省 90% 输入 token）—— 缓存系统 prompt 和知识库 chunk
- 长对话用摘要而非完整历史
- 固定信息（学校 deadline）直接查库，不让 LLM 生成
- 意图识别先走便宜模型，复杂问题再升级

### 7.4 计费模式

| 套餐 | 价格 | 包含 | 目标用户 |
|---|---|---|---|
| 免费版 | 0 | 每日 20 条 / 单会话基础 skill | 拉新 |
| Pro 订阅 | 199/月 或 1599/年 | 全 skill 无限 + 案例库访问 + 进度管理 | DIY 申请者主力 |
| Premium 一次性 | 4999/季 | Pro + 真人导师 2 小时 + 文书深度精修 | 高需求用户 |
| B 端 SaaS | 199/席位/月 | 中介顾问 Copilot 工具 | 中小留学机构 |

### 7.5 单位经济（保守估算）

| 指标 | 数值 |
|---|---|
| DIY 申请者 LTV | 约 1500-1800 元（一个申请季） |
| API 成本（路由优化后） | 约 80-150 元/活跃用户/季 |
| 毛利率 | 约 80% |
| 对标传统中介客单价 | 30,000-100,000 元 |
| 价值替代比 | 约 1/50 - 1/30 |

### 7.6 合规底线

- 国内运营：算法备案 + 生成式 AI 备案（必须早做，否则上线即下架）
- 国产模型做兜底通道：网络故障 / 海外模型限流时切换
- 所有 prompt 抽象到配置层，模型调用接口统一封装，一周内能切换底层模型

## 八、MVP 范围（v0.3 上线版）

### 8.1 v0.3 上线只做什么

聚焦「硕士 DIY · 美国主战场 · 学校信息查询 + 选校 + 文书思路」。把核心交互 C3 + 4-6 个核心 skill 做透。

| 优先级 | Skill / 功能 | 上线理由 |
|---|---|---|
| P0 | 项目信息查询（Pattern A） | 首问通常是「这个学校 / 项目什么情况」 |
| P0 | 录取要求解读（Pattern A） | 高频 + 高粘性 + 完美演示 C3 交互 |
| P0 | C3 双栏 + 划词解读 + URL 粘贴 | 产品的灵魂时刻 |
| P0 | 背景评估（顾问型） | 用户首问通常是「我能申到什么」 |
| P0 | 选校匹配（顾问型） | 决策 anchor |
| P0 | PS 思路（顾问型，不动笔） | 高频、合规 |
| P1 | 案例库 MVP（Pattern C） | 信任建立 + 留存 |
| P1 | 申请时间线（顾问型） | 陪伴感 |
| P1 | USCIS / 签证 Pattern A | 季节性高峰流量 |

### 8.2 v0.3 不做什么

- 本科申请 skill（学生 / 家长决策链复杂，留到 v0.4）

- Chrome 插件（重型工程，留到 v1.0）
- 社区 / UGC（冷启动重内容轻社区）
- 移动 App（先 Web 验证 PMF）
- B 端 SaaS（C 端 PMF 优先）

## 九、风险与应对

| 风险 | 可能性 | 应对策略 |
|---|---|---|
| 通用 AI 浏览器（Comet / Atlas）下沉到留学 | 高 | 用 6-12 个月构建数据护城河（案例 + 知识库 + 中国学生本地化） |
| Claude / GPT 直接进入中国市场 | 中 | 我们的价值不在「用 Claude」，而在垂直化 + 中国学生本地化。即便他们进来，垂直深度依然是壁垒 |
| 学校官网反爬 / 法律风险 | 中 | T1 数据用合规途径（API、RSS）；T2 用 Reader 模式（transformative use）；保留每个学校的「↗ 在原网站打开」 |
| AI 政策收紧（学校禁止 AI 写文书） | 中 | 我们本来就不写文书，只给思路。反而站到合规一边，可以做「AI 合规检测」反向卖点 |
| API 涨价 / 限流 | 中 | 多模型路由 + Prompt Cache + 国产模型兜底，单一供应商风险已分散 |
| 冷启动数据不足 | 高 | 前 100 用户人工兜底 + 持续抓取 + 用户上传案例换额度 |
| 国内 AI 备案延迟 | 高 | 立刻启动备案；备案期间用海外节点服务海外华人 / 港新用户 |
| 传统中介反扑（收购 / 模仿） | 中 | 建立用户口碑，让「AI 替代中介」成为认知；早期不与机构合作 |

## 十、附录

### 10.1 关键术语

| 术语 | 含义 |
|---|---|
| C3 | 我们的核心交互模型：左侧官网原文 + 右侧 AI 顾问 + 划词解读 |
| Pattern A/B/C/D | 左侧四种内容形态：官网阅读 / 多源对照 / 案例库 / 表单截图 |
| 三圈（绿/黄/红） | 回答边界：留学专属 / 留学场景 / 完全无关 |
| Reader / 原版 | v0.2 探索过的双视图，最终在 C3 简化掉了 |
| Skill Router | 意图识别 + 自动调用对应 skill 的编排层 |
| Prompt Cache | Claude 的 prompt 缓存特性，省 90% 输入 token |
| RAG | Retrieval Augmented Generation，事实型答案查库不靠记忆 |

### 10.2 已废弃 / 推迟的方向

- AI 留学浏览器（fork Chromium）—— 太重，让位给 Web + Chrome 插件路线
- v0.1 的「AI 替你做留学」定位 —— 替换为「AI 帮你看懂留学」
- v0.2 段落级双语视图 —— 用户感觉太复杂，简化为 C3 纯原文
- Reader / 原版 双视图切换 —— 学生易疑「原版是假的」，砍掉
- AI 代写文书 —— 合规风险高 + 违背「AI 不动笔」原则

### 10.3 决策记录链

本文档不是凭空写出，是过去几周设计迭代的最终态。重要决策的演化：

- **D1**：从「AI 留学浏览器」收敛到「Web + 对话框」形态
- **D2**：从「AI 替学生做事」切换到「AI 帮学生看懂」
- **D3**：从「通用顾问」加入「官网解读」双能力
- **D4**：UI 从侧边栏 → 全屏对话 → C 双栏 → C2 双语 → C3 纯原文（最终）→ 见 `decisions/0001`
- **D5**：原版 toggle 砍掉，简化为纯 Reader + ↗ 在原网站打开
- **D6**：技能包从「按用户旅程」重构为「按左侧 Pattern」
- **D7**：三圈边界模型定型，明确产品克制
- **D8**：品牌名定为 izanoffer → 见 `decisions/0002`
